# รายงานการวิเคราะห์โครงสร้างหน้าบันทึกค่าใช้จ่าย (PEAK Expense Entry)

**Target URL:** `https://secure.peakaccount.com/expense/purchaseInventory`

จากการให้บอทเข้าไปสำรวจหน้าเว็บไซต์ พบโครงสร้างหลักและองค์ประกอบสำคัญสำหรับการเขียน Bot Automation ดังนี้:

### 1. ส่วนหัวเอกสาร (Header Section)

- **ชื่อหน้า:** "บันทึกซื้อสินค้า/บริการ"
- **เลขที่ใบกำกับภาษี:** ช่อง Input (ID: `t250xg` หรือคล้ายกัน แต่ละ session อาจแรนดอม ID แนะนำให้ใช้ Selector แบบเจาะจง `placeholder` หรือ `aria-label`) - สำหรับระบุเลขที่จากใบเสร็จ/ใบกำกับภาษีของผู้ขาย
- **เลขที่เอกสาร:** ช่อง Input - โดยปกติระบบจะเจนให้ล่วงหน้า (เช่น `EXP-20260300001`)

### 2. ข้อมูลผู้ขาย (Vendor Details)

- **ชื่อผู้ขาย:** ช่องค้นหา/Dropdown - ใช้พิมพ์ชื่อเพื่อเลือกผู้ติดต่อที่มีอยู่ หรือสร้างใหม่
- **วันที่ออก:** ช่องเลือกวันที่ - วันที่ในใบเสร็จ
- **วันที่ครบกำหนด:** ช่องเลือกวันที่ - วันที่ต้องชำระเงิน

### 3. ข้อมูลราคาและภาษี (Price and Tax Settings)

- **ประเภทราคา:** Dropdown - ตัวเลือกหลักคือ "แยกภาษี" (VAT Excluded), "รวมภาษี" (VAT Included), หรือ "ไม่มีภาษี" (No VAT)

### 4. ตารางรายการ (Line Items Table)

รายการสินค้าหรือบริการแต่ละบรรทัดมีช่องกรอกข้อมูลดังนี้:

- **สินค้า/บริการ:** Dropdown - ค้นหาจากฐานข้อมูลสินค้า
- **บัญชี:** Dropdown - สำหรับเลือกผังบัญชี (Chart of Accounts)
- **จำนวน:** Input - ค่าเริ่มต้นคือ `1.00`
- **ราคา/หน่วย:** Input - ค่าเริ่มต้นคือ `0.00`
- **ส่วนลด/หน่วย:** Input - ค่าเริ่มต้นคือ `0.00`
- **ภาษี:** Dropdown - ค่าเริ่มต้นคือ `7%`
- **หัก ณ ที่จ่าย (WHT):** Dropdown - ตัวเลือกได้แก่ 1%, 3%, 5% หรือไม่ระบุ
- **คำอธิบายรายการ:** Textarea - สำหรับระบุรายละเอียดเพิ่มเติมของรายการนั้นๆ

### 5. ส่วนสรุปและปุ่มดำเนินการ (Footer & Actions)

- **ยอดเงิน:** แสดงส่วนลดรวมและจำนวนเงินทั้งสิ้นในกล่องสีน้ำเงินเข้ม
- **ปุ่มดำเนินการ:**
  - **ยกเลิก:** ปุ่มสีเทาด้านซ้าย
  - **บันทึกร่าง:** ปุ่มสีน้ำเงิน
  - **อนุมัติบันทึกซื้อสินค้า:** ปุ่มสีเขียว - สำหรับยืนยันรายการลงบัญชีจริง

# 🤖 ขั้นตอนการทำงานของบอทบันทึกค่าใช้จ่าย PEAK

> ไฟล์อ้างอิง: `backend/routes/bot-automation.js`

---

## ขั้นตอนที่ 1: เตรียมระบบ

### 1.1 รับคำสั่งจาก API

- Frontend ส่ง `POST /start` พร้อม `profileId` + `excelPath`
- ระบบสร้าง Job ใหม่ สถานะ = `queued`

### 1.2 ตรวจสอบคิว

- ถ้างานที่กำลังรันอยู่ **< 5** → เริ่มทำทันที
- ถ้างานที่กำลังรันอยู่ **≥ 5** → เข้าคิวรอ (FIFO)

### 1.3 อ่านไฟล์ Excel

- อ่าน 3 Sheets:
  - `มีภาษีมูลค่าเพิ่ม` — รายการค่าใช้จ่ายที่มี VAT
  - `ไม่มีภาษีมูลค่าเพิ่ม` — รายการค่าใช้จ่ายที่ไม่มี VAT
  - `ที่อยู่แต่ละบริษัท` — ข้อมูลที่อยู่ผู้ขาย
- รวม Sheet 1 + 2 เป็น `allTransactions`
- จัดกลุ่มรายการที่มี `เลขที่เอกสาร` เดียวกัน → อยู่ใน **บิลเดียวกัน**

### 1.4 เปิด Browser

- ใช้ Chromium (Playwright) แบบ headless: false
- ใช้ Browser Instance เดียวร่วมกันทุก Job (ประหยัด RAM)
- แต่ละ Job สร้าง Context แยก (Isolation)

## ขั้นตอนที่ 2: Login เข้าระบบ PEAK

### 2.1 เปิดหน้า Login

- เข้า `https://secure.peakaccount.com/`
- รอฟอร์ม Login โหลด (timeout 15 วิ)

### 2.2 กรอกข้อมูล

- กรอก **อีเมล** จาก Bot Profile
- กรอก **รหัสผ่าน** (decrypt จาก DB ด้วย AES-256-CBC)
- กดปุ่ม **"เข้าสู่ระบบ PEAK"**

### 2.3 ตรวจสอบผลลัพธ์

- ถ้า URL มี `/home` หรือ `/selectlist` → ✅ Login สำเร็จ
- ถ้า URL อื่น → ⚠️ แจ้งเตือนแต่ทำต่อ

### 2.4 เข้าหน้าบริษัท

- ใช้ `peakCode` ไปที่ `https://secure.peakaccount.com/home?emi={peakCode}`
- ถ้าไม่มี `peakCode` → ❌ หยุดทำงานทันที

## ขั้นตอนที่ 3: ค้นหาและเลือกผู้ขาย (Vendor)

> ทำซ้ำทุกบิล — เริ่มต้นด้วยการรีเฟรชหน้าสร้างบิลใหม่ทุกครั้ง

### 3.1 ตรวจสอบเลขภาษี

- ถ้า `เลขประจำตัวผู้เสียภาษี` ว่าง → ⚠️ ข้ามบิลนี้ทั้งบิล

### 3.2 ค้นหาผู้ขาย

- คลิกช่อง "พิมพ์เพื่อค้นหาผู้ติดต่อ"
- พิมพ์เลขภาษี 13 หลัก
- รอ API ดึงรายชื่อผู้ขาย

### 3.3 วิเคราะห์ผลลัพธ์ Dropdown

| เงื่อนไข                        | การทำงาน             |
| ------------------------------- | -------------------- |
| พบผู้ขาย + **สาขาตรงกับ Excel** | ✅ คลิกเลือก         |
| พบผู้ขาย + **สาขาไม่ตรง**       | กดเพิ่มผู้ติดต่อใหม่ |
| **ไม่พบผู้ขายเลย**              | กดเพิ่มผู้ติดต่อใหม่ |

### 3.4 การจับคู่สาขา

- Excel `สาขา` = ว่าง / `00000` / `สำนักงานใหญ่` → Match กับ Option ที่มี "สำนักงานใหญ่"
- Excel `สาขา` = `00069` → Match กับ Option ที่มี "00069"

### 3.5 สร้างผู้ติดต่อใหม่ (ถ้าจำเป็น)

1. เปิด Modal "เพิ่มผู้ติดต่อ"
2. กรอกเลขภาษี 13 หลัก (แยกทีละช่อง `input.inputId` x 13)
3. เลือกสาขา:
   - `00000` → เลือก "สำนักงานใหญ่"
   - อื่นๆ → เลือก "สาขา" แล้วกรอก 5 หลัก (ช่อง 14-18)
4. กดปุ่ม **ค้นหา** (API กรมพัฒน์)
5. กรอกที่อยู่ (ถ้ามีใน Sheet `ที่อยู่แต่ละบริษัท`)
   - ถ้า PEAK ว่าง → กรอกทับ
   - ถ้า PEAK มีอยู่แล้ว → เปรียบเทียบ (normalize) ถ้าต่าง ≥ 20% → กรอกทับ
   - ฟอร์มที่อยู่ถูกย่อ → กดปุ่ม "ย่อ/ขยาย" กางออกก่อน
6. กดปุ่ม **"เพิ่ม"** บันทึก

## ขั้นตอนที่ 4: กรอกข้อมูลบิลและรายการสินค้า

### 4.1 ข้อมูลหลัก (ทำครั้งเดียวต่อบิล)

| ฟิลด์             | คอลัมน์ Excel  | รูปแบบรองรับ                               |
| ----------------- | -------------- | ------------------------------------------ |
| วันที่ออก         | `วันที่`       | Excel serial / `YYYY-MM-DD` / `DD/MM/YYYY` |
| เลขที่ใบกำกับภาษี | `เลขที่เอกสาร` | ข้อความ                                    |

### 4.2 วนลูปรายการสินค้า (ทำทุกแถวในบิล)

> ถ้าเป็นรายการที่ 2+ → กดปุ่ม **"เพิ่มรายการใหม่"** ก่อน

#### 4.2.1 บัญชี/ค่าใช้จ่าย (`โค้ดบันทึกบัญชี`)

- หา Input ด้วย 3 วิธี (Fallback):
  1. `input[placeholder*="บัญชี/ค่าใช้จ่าย"]`
  2. `input[placeholder*="รหัสบัญชี"]`
  3. `.textSelectedDropdown`
- คลิกเปิด Dropdown → พิมพ์โค้ด
- เงื่อนไข:
  - พบตรงกัน → ✅ เลือก
  - มีแต่ไม่ตรง → ⚠️ เลือกตัวแรก
  - มีแค่ "เพิ่มบัญชีใหม่" → ❌ ข้าม

#### 4.2.2 ประเภทราคา (เฉพาะรายการแรกของบิล)

- ตั้งเป็น **"รวมภาษี"** เสมอ
- ค้นหา Multiselect ที่มี label "แยกภาษี"/"รวมภาษี"/"ไม่มี"

#### 4.2.3 ราคา/หน่วย (`ยอดหลังบวกภาษีมูลค่าเพิ่ม`)

- กรอกลง `#inputPrice input` (ตัวสุดท้าย)
- วิธี: Ctrl+A → Backspace → Type → Tab

#### 4.2.4 ภาษี VAT (`ยอดภาษีมูลค่าเพิ่ม`)

| ยอดจาก Excel | เลือกใน Dropdown |
| ------------ | ---------------- |
| > 0          | **7%**           |
| = 0 หรือว่าง | **ไม่มี**        |

- Element: `#dropdownTax` (ตัวสุดท้าย)

#### 4.2.5 หัก ณ ที่จ่าย (`หักณที่จ่าย`)

| ค่าจาก Excel   | เลือกใน Dropdown |
| -------------- | ---------------- |
| มีค่า เช่น `5` | **5%**           |
| ว่าง           | ข้ามไม่ทำ        |

- Element: `#taxWithheld .whtDropdown` (ตัวสุดท้าย)

## ขั้นตอนที่ 5: เลือกรูปแบบชำระเงิน + อนุมัติ

### 5.1 กดเลือก "ขั้นสูง"

- เลื่อนลงล่างสุด
- คลิก label "ขั้นสูง"

### 5.2 ตรวจสอบ `โค้ดตัดชำระเงิน`

| ค่าใน Excel                | ตรวจสอบ           | การทำงาน                                                                                 |
| -------------------------- | ----------------- | ---------------------------------------------------------------------------------------- |
| **ว่างเปล่า**              | -                 | คลิก **"ยังไม่ชำระเงิน (ตั้งหนี้ไว้ก่อน)"**                                              |
| **ตัวเลข** (เช่น `212201`) | `/^\d+$/` = true  | ติ๊ก "ค่าธรรมเนียม หรือรายการปรับปรุง" → กรอกลง `#DropdownPaymentChartOfAccount` → เลือก |
| **ตัวอักษร** (เช่น `SCB`)  | `/^\d+$/` = false | กรอกลง `#DropdownPaymentBankAccount` → เลือก                                             |

### 5.3 อนุมัติบันทึกค่าใช้จ่าย

1. กดปุ่ม **"อนุมัติบันทึกค่าใช้จ่าย"** (`div.button.mint p`)
2. รอเลขเอกสารใหม่ปรากฏ (เช่น `#EXP-20260100001`) — timeout 15 วิ
3. ดึงเลขเอกสาร → ลบ `#` ออก → ได้ `EXP-20260100001`

## ขั้นตอนที่ 6: ตรวจสอบและแก้ไขยอด VAT

### 6.1 คำนวณ Expected VAT

```
expectedVat = ผลรวมยอดภาษีมูลค่าเพิ่ม จากทุกแถวในบิลเดียวกัน
```

### 6.2 ดึง Actual VAT จากหน้าเว็บ

- ใช้ `page.evaluate()` อ่าน `document.body.innerText`
- หาบรรทัด "ภาษีมูลค่าเพิ่ม" จากล่างขึ้นบน
- ดึงตัวเลขสุดท้าย (ข้ามค่า `7` ที่มาจาก "7%")

### 6.3 เปรียบเทียบ

| ผลลัพธ์   | เงื่อนไข | การทำงาน          |
| --------- | -------- | ----------------- | ------- | --------------------- |
| ✅ ตรงกัน | `        | expected - actual | ≤ 0.05` | จบบิลนี้ → วนบิลถัดไป |
| ⚠️ ไม่ตรง | `        | expected - actual | > 0.05` | เข้าโหมดแก้ไข ↓       |

### 6.4 โหมดแก้ไขภาษี (เมื่อ VAT ไม่ตรง)

1. กดปุ่ม **"ตัวเลือก"** (`div.buttonNotDefaultOption p`)
2. เลือก **"แก้ไข"** (`div.optionBox div.option`)
3. รอหน้าเว็บโหลดเสร็จ (`waitForLoadState`)
4. หาไอคอน ✏️ (`i.fa-pen.cursor, #iconClick`) — timeout 30 วิ
5. คลิกไอคอนเพื่อเปิดช่องแก้ไข
6. กรอก **ยอดภาษีที่ถูกต้อง** ลง `input[placeholder="0.00"]`
7. กด **Enter**
8. **คลิกพื้นที่ว่าง** (`page.mouse.click(10, 500)`) — เพื่อ trigger Blur ให้ Vue บันทึกค่า
9. รอ **1 วินาที** ให้ Vue อัปเดต
10. เช็ค **"ยังไม่ชำระเงิน (ตั้งหนี้ไว้ก่อน)"** ปรากฏอยู่ไหม → ถ้ามีให้คลิกก่อน
11. กดปุ่ม **"บันทึก"** (`div.purple.textblack button`)
12. รอระบบประมวลผล (10 วิ dev mode)

## ระบบ Error Handling & Recovery

| สถานการณ์            | การทำงาน                                                     |
| -------------------- | ------------------------------------------------------------ |
| Page ถูกปิดจากภายนอก | ตรวจจับ `page.isClosed()` → เปิดหน้าใหม่จาก context          |
| โหลดหน้าเว็บไม่ขึ้น  | ข้ามไปทำบิลถัดไป                                             |
| ไม่มีเลขภาษี         | ข้ามบิลทั้งบิล                                               |
| เกิด Error ภายในบิล  | 📸 แคปหน้าจอ → บันทึกที่ `backend/screenshots/` → ทำบิลถัดไป |
| Job Error (Fatal)    | ปิด Context → ดึงงานถัดไปจากคิว                              |

## API Endpoints

| Method | Path                   | คำอธิบาย                            |
| ------ | ---------------------- | ----------------------------------- |
| `GET`  | `/excel-files?dir=...` | รายชื่อไฟล์ Excel จาก directory     |
| `POST` | `/start`               | สั่งเริ่ม Bot (ต้องส่ง `profileId`) |
| `GET`  | `/jobs`                | ดูรายการ Job ทั้งหมด                |
| `GET`  | `/logs/:jobId`         | ดู Log ของ Job ที่ระบุ              |
| `GET`  | `/stream/:jobId`       | SSE Stream Log แบบ Real-time        |
| `POST` | `/stop/:jobId`         | หยุด Job + ปิด Browser              |
